## 1. 内存与上下文技巧 (Memory & Context Skills)

- 分层记忆架构 (Tiered Memory)
    
    - 短期记忆：直接将当前对话的最后数轮 `Messages` 传入上下文。
    - 长期记忆：使用外部向量数据库存储历史对话。
    - 语义总结记忆：使用一个低成本模型（如 GPT-4o-mini），定期将过往对话压缩为一小段“用户画像与核心事实”摘要，作为系统提示词（System Prompt）常驻。
    
- 动态上下文修剪 (Context Window Pruning)
    
    - 在长对话中，使用 Token 计数器。
    - 当接近模型限制时，优先剔除冗长的工具执行原始日志（Tool Outputs）。
    - 仅保留工具执行的最终结论（Derived Insights）。
    

## 2. 工具与 API 技能 (Tool Use & Function Calling)

- 工具描述工程 (Tool Description Engineering)
    
    - 大模型靠 API 的 `description` 决定是否调用该工具。
    - 技巧：在描述中明确写入调用场景与参数限制（例如：“_当用户询问天气且明确提及城市时调用此工具。必须传入标准的拼音城市名，如 beijing_”）。
    
- 工具参数强校验 (Schema Validation)
    
    - 使用 Pydantic 或 JSON Schema 严格定义工具的入参。
    - 如果大模型生成的参数缺失或格式错误，直接拦截请求，不调用真实 API。
    
- 自动错误修正循环 (Auto-Correction Loop)
    
    - 当工具执行报错（如 API 返回 400 错误）时，不要将报错抛给用户。
    - 将报错信息（Error Message）重新包装成一条 `User Message` 发回给大模型。
    - 提示词示例：“_你上一次生成的工具参数导致了以下错误：[报错内容]。请重新修正参数并再次调用。_”
    

## 3. 控制流与工程化技巧 (Flow Control & Guardrails)

- 状态机控制 (State Machine Over Autonomy)
    
    - 放弃让大模型“完全自由决定下一步”。
    - 使用 LangGraph 等框架，将业务流程预设为硬编码的图结构（Graph）或状态机（State Machine）。
    - 大模型仅在特定的节点（Node）内做多选题或填空题，确保行为 100% 可预测。
    
- 输出护栏 (Output Guardrails)
    
    - 在 Agent 输出最终结果前，架设一层“合规/格式审查 Agent”。
    - 技巧：使用小模型进行快速分类，检查输出中是否包含敏感词、是否符合业务规范、或者是否打破了预设的 JSON 格式。
    
- 确定性回退机制 (Fallback Mechanism)
    
    - 为 Agent 设置最大尝试次数（通常为 3 次）。
    - 如果 Agent 在多次反思或工具调用中陷入死循环或持续报错，触发硬编码的回退逻辑（例如：直接输出预设的友好提示，或转人工客服）。
