## 🔍 针对性学习重点（实战化建议）

## 1. 虚拟线程的“新规矩”

AI 可能还会教你用 `ThreadLocal` 来池化对象。

- 学习重点：在 Java 21 中，虚拟线程是廉价的，不再需要池化。你需要学习 Scoped Values (孵化中) 或重构代码习惯，避免在数百万虚拟线程中滥用 `ThreadLocal` 导致内存膨胀。

## 2. “防御式”模式匹配

AI 写的 `instanceof` 和强制转型既丑又易出错。

- 学习重点：熟练掌握 Java 17+ 的模式匹配 (Pattern Matching)。
- 应用：当 AI 生成复杂的条件判断时，你应强制要求其使用 `switch` 表达式。这样如果漏掉一种业务情况（Case），编译器能直接报错。

## 3. 并发安全审计陷阱

AI 最容易犯的错误是：“看起来线程安全，实际不安全”。

- 学习案例：AI 可能会写 `if (!map.containsKey(key)) { map.put(key, value); }`。
- 你需要学：掌握 `computeIfAbsent` 等原子操作，一眼看出这种“先检查后执行”的逻辑在并发下会失效。

## 4. 资源治理 (Try-with-resources)

- 学习重点：Java 9+ 改进的资源管理。
- 验证点：AI 生成的 IO 流、JDBC 连接、HttpClient 调用，必须严格审查是否有资源释放逻辑，否则在高频触发下系统会迅速挂掉。

## 💡 学习建议：如何“练兵”？

不要自己写代码，尝试“喂给 AI 一个错误命题”：

> _“给我写一个 Java 高并发单例模式，并在其中执行一个耗时的 IO 操作，使用旧的线程池方案。”_

然后，你尝试用 Java 21 虚拟线程 和 双重检查锁/枚举单例 去重构它、挑它的刺。这种“找茬式学习”是目前 Java 开发者提升内核能力最快的方式。
