为了帮您构建最完整的 GitHub Copilot CLI 命令全景图，这里为您将所有命令按照“外层原生 Shell 命令（客户端管理）”与“交互式斜杠命令（大模型协同）”进行分类整合，并详细标注每个命令的功能、参数以及它们与大模型的关联性。

## 一、 客户端管理：外层原生 Shell 命令（模型无关）

这类命令直接在您的本地终端（如 Bash、Zsh、PowerShell）中输入，用于管理本地客户端、进行全局配置或单次指定模型，不需要大模型参与计算。

|功能分类|原生 Shell 命令 / 参数|核心功能说明|进阶使用技巧与参数|
|---|---|---|---|
|版本与更新|`copilot --version`|打印当前本地安装的 CLI 客户端具体版本号。|排查新功能是否支持、或向官方提 Bug 时首要提供的数据。|
||`copilot update`|强制检查并拉取最新发布的版本进行覆盖安装。|客户端默认会在启动时静默检查，此命令用于手动强制升级。|
|基础功能|`gh copilot suggest "[需求]"`|询问并推荐符合您自然语言描述的 Shell 命令。|示例：`gh copilot suggest "查找当前目录下所有大于 100M 的文件"`|
||`gh copilot explain "[命令]"`|详细剖析并解释一段复杂的终端原生命令。|示例：`gh copilot explain "iptables -A INPUT -p tcp --dport 80 -j ACCEPT"`|
|初始化与配置|`gh copilot config`|进入交互式菜单，配置默认 Shell、数据收集偏好等。|首次安装或需要更改默认行为（如从 Bash 换到 Zsh）时使用。|
|模型指定参数|`--model <model-name>`|单次覆盖模型： 执行当前命令时，强行指定使用某特定大模型。|示例：`gh copilot suggest "高级命令" --model claude-3-5-sonnet`|
||`export COPILOT_MODEL=...`|全局环境变量： 将当前终端会话的默认大模型固定为指定型号。|写入 `.bashrc` 或 `.zshrc` 可永久生效，无需每次在交互内切换。|
|日志与调试|`--log-level <level>`|调整输出的日志级别（如 `debug`, `info`, `warn`, `error`）。|当命令报错或 Agent 卡死时，开启 `debug` 级别查看具体 API 报错。|

这类命令必须在运行 `copilot` 进入 AI 交互会话后，在聊天输入框内键入。高级命令的执行成败高度依赖底层大模型的推理能力。

## 💡 记忆与使用心法

- 什么时候不用斜杠？ 凡是用来维护客户端本身（升级、查版本、改本地设置），或者只做单次单条命令查询时，直接在终端敲原生命令（如 `copilot update`）。
- 什么时候必须用斜杠？ 凡是已经执行了 `copilot` 命令、进到了 AI 的聊天交互界面后，所有对 AI 发出的指令和环境控制，必须加斜杠（如 `/agent`、`/model`）。
