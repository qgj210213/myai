## 🚀 Cursor 核心操作手册

| 类别       | 简要 (Brief) | 用途              | 操作方式           | 快捷键 (Win/Mac)                |
| -------- | ---------- | --------------- | -------------- | ---------------------------- |
| 智能代码     | Edit       | 在当前代码行生成或修改代码   | 在编辑器内输入指令      | `Ctrl` / `Cmd` + `K`         |
| 对话答疑     | Chat       | 针对全库或特定代码提问     | 在侧边栏与 AI 对话    | `Ctrl` / `Cmd` + `L`         |
| 全案构建     | Composer   | 跨文件协同，构建复杂功能    | 呼出大窗口模式进行开发    | `Ctrl` / `Cmd` + `I`         |
| 采纳修改     | Accept     | 接受 AI 提出的所有修改建议 | 预览后确认应用到文件     | `Ctrl` / `Cmd` + `Enter`     |
| 舍弃修改     | Reject     | 拒绝或撤销 AI 的修改建议  | 放弃本次生成的变更      | `Ctrl` / `Cmd` + `Backspace` |
| 上下文管理    | Mention    | 引入特定文件、文档或代码块   | 在对话框内输入 `@` 符号 | `输入 @`                       |
| 预测补全     | Tab        | 行内自动预测并补全后续代码   | 看到灰色提示时按键采纳    | `Tab`                        |
| 终端增强<br> | Term K     | 用自然语言生成终端命令     | 在终端面板内输入指令     | `Ctrl` / `Cmd` + `K` (终端内)   |
|          |            |                 |                |                              |

---

小贴士：

- Composer (Cmd+I) 是目前 Cursor 最强大的功能，适合处理需要同时修改好几个文件的复杂任务。
- 如果你觉得 AI 不够了解你的项目，试试在 Chat 里输入 @Codebase 进行全库扫描。
